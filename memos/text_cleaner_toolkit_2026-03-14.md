## Profile System

Each corpus type gets a YAML profile defining:
- Line format (anchored, plain, numbered)
- Protected zones (frontmatter, headers)
- Domain-specific allowlist path
- Which single-letter prefixes to scan for fusion
- Context-sensitive multi-word patterns
- False positive word lists

Three profiles ship out of the box. New corpora just need a YAML file.

## Workflow for New Texts

1. Import text via Docling
2. Choose/create a profile
3. `clean.py --dry-run` → see mechanical errors
4. `clean.py --apply` → auto-fix P1-P5
5. `scan.py` → find fused footnote markers
6. `fix.py --generate-map` → draft replacement map from scan output
7. Review map (Claude or human classifies each token)
8. `fix.py --map` → apply curated fixes
9. Audit memo + validation

## Key Insight Preserved

The aspell-gated detection approach (only flag tokens aspell rejects where
the remainder IS recognized) eliminates ~97% of false positives. All three
scripts use this technique.
